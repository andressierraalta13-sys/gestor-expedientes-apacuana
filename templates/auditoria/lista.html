Guía: Auditoría de Movimientos

history 1. ¿Qué es esta sección?

Es la bitácora histórica del sistema. Aquí queda registrado, de forma permanente e inalterable, cada evento ocurrido en la base de datos sobre los módulos de Calificaciones y Expedientes. Sirve para garantizar la trazabilidad y saber en todo momento quién hizo qué y cuándo.

table_rows 2. Cómo leer la tabla

Cada fila representa un evento registrado. Las columnas muestran:

  • Fecha y Hora: momento exacto en que ocurrió la acción.
  • Usuario / IP: persona responsable del movimiento.
  • Acción Detectada: INSERT (se creó un registro), UPDATE (se modificó) o DELETE (se eliminó).
  • Módulo Alterado: sección del sistema afectada.
  • Detalle del Registro: descripción específica del cambio realizado.

filter_alt 3. Filtrar y buscar

Utilice el campo "Filtrar por usuario..." para localizar las acciones de una persona en particular, o el botón "Filtrar Fechas" para acotar el historial a un período de tiempo específico.

delete_forever 4. Limpiar la bitácora

El botón rojo "Limpiar" solo está disponible para el rol Desarrollador y vacía por completo el historial de auditoría. Esta acción es irreversible: una vez confirmada, los registros no se pueden recuperar. Úsela con extrema precaución.
